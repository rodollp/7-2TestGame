# MegaBonk Clone Project

## 현재 구현

### Player

- Input System을 이용한 이동
- 카메라 방향 기준 이동
- Rigidbody를 이용한 물리 이동

---

### MonsterData (ScriptableObject)

몬스터의 설계도 역할

- 이름
- 최대 체력
- 공격력
- 이동속도
- 경험치
- 골드
- 기타 기본 능력치

실제 게임에서는 직접 사용하지 않고 MonsterStatus가 값을 복사하여 사용한다.

---

### MonsterStatus

런타임에서 사용하는 몬스터 정보

MonsterData의 값을 Awake에서 복사한다.

관리하는 데이터

- CurrentHp
- MaxHp
- MoveSpeed
- AttackPower
- Exp
- Gold
- Name

사망 여부 관리

- TakeDamage()
- Die()

향후

- OnDead 이벤트 발생

---

### MonsterAI

NavMeshAgent를 이용한 플레이어 추적

기능

- Player 탐색
- NavMeshAgent 초기화
- MonsterStatus의 MoveSpeed 적용
- 플레이어 위치를 실시간으로 따라감

FSM은 사용하지 않음.

---

### MonsterContactDamage

근접 몬스터 전용 공격

플레이어와 Collision 중일 때

- AttackCooldown마다
- AttackPower만큼 데미지

메가봉크 스타일의 충돌 공격 방식

---

## 현재 구조

MonsterData
↓
MonsterStatus
↓
MonsterAI
MonsterContactDamage

---

## 설계 원칙

### Data

ScriptableObject

- 설계도
- 수정하지 않음

### Status

런타임 데이터

- Data 복사
- 실제 게임에서 사용하는 값
- 버프/디버프 적용

### AI

행동만 담당

- 플레이어 추적
- 이동

### Damage

데미지만 담당

- 충돌 공격
- 쿨타임 관리
