# 2026-07-13 개발 일지

## 프로젝트 목표

투사체 공격 시스템을 기존 무기 시스템에 연결하고, 공격 구조를 점검 및 리팩터링하였다.

---

# 오늘 구현한 내용

## 1. Projectile 구현

투사체가 실제로 씬에서 이동하고 충돌하는 기능을 구현하였다.

### 역할

- 이동 방향 저장
- 이동 속도 적용
- Rigidbody를 이용한 이동
- 일정 시간이 지나면 비활성화
- 적과 충돌 시 데미지 적용

Projectile는 화살 전용 스크립트가 아니라 앞으로 추가될 모든 투사체(화살, 마법탄, 총알 등)의 공통 동작을 담당하도록 설계하였다.

---

## 2. ProjectileSpawner 분리

Projectile 생성 책임을 AttackStrategy에서 분리하였다.

### 현재 구조

ProjectileAttack

↓

ProjectileSpawner

↓

Projectile 생성

↓

Projectile.Init()

Spawner 내부에서는 현재 Instantiate를 사용하지만 이후 Object Pool로 교체할 예정이다.

---

## 3. AttackContext 구성

공격 전략에서 필요한 객체를 하나로 전달하기 위해 AttackContext를 구성하였다.

```csharp
public class AttackContext
{
    public PlayerAttack PlayerAttack { get; }
    public ProjectileSpawner ProjectileSpawner { get; }
}
```

AttackContext는 공격에 필요한 시스템을 전달하는 객체이며, 공격 전략이 필요한 기능만 사용할 수 있도록 구성하였다.

---

## 4. TargetFinder 분리

자동 공격 내부에 있던 타겟 탐색을 별도 클래스로 분리하였다.

현재 역할

- 살아있는 몬스터 탐색
- 사거리 체크
- 가장 가까운 몬스터 반환

```csharp
public IDamageable FindNearestTarget(Vector3 origin, float range)
```

현재는

```csharp
FindObjectsByType<MonsterStatus>()
```

를 이용하여 몬스터를 검색하고 있다.

---

## 5. Projectile 구조 점검

Projectile는 앞으로 다양한 무기에서 공통으로 사용할 수 있도록 설계하였다.

현재 Projectile의 책임

- 이동
- 충돌
- 수명 관리
- 데미지 전달

Projectile가 공격력을 계산하지 않고 PlayerAttack을 이용하여 데미지를 적용하는 구조를 유지하였다.

---

# 오늘 배운 점

- Projectile는 화살이 아니라 모든 투사체의 공통 객체이다.
- ProjectileSpawner를 분리하면 이후 Object Pool 적용이 쉬워진다.
- AttackContext는 공격에 필요한 시스템을 하나로 전달하기 위한 객체이다.
- TargetFinder는 공격 실행이 아니라 타겟 탐색만 담당해야 한다.
- 현재 FindObjectsByType은 임시 구현이며 이후 MonsterRegistry로 교체할 예정이다.
