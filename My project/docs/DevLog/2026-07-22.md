# Dev Log - 2026-07-22

## 오늘 목표

게임의 메인 루프를 리팩토링하여
각 시스템의 책임을 명확하게 분리한다.

---

# 완료한 작업

## 1. GameTimer 리팩토링

기존에는 MonsterSpawnSystem과 GameTimer가
각각 플레이 시간을 관리하고 있었다.

### 변경 전

- GameTimer
    - UI 출력
    - 플레이 시간 계산

- MonsterSpawnSystem
    - 플레이 시간 계산
    - 몬스터 소환

→ 동일한 게임 시간을 두 곳에서 관리

### 변경 후

GameTimer가 플레이 시간을 단독으로 관리하도록 변경하였다.

추가한 기능

- PlayTime 프로퍼티
- IsStageTimeEnded 프로퍼티
- OnStageTimeEnded 이벤트

MonsterSpawnSystem은
GameTimer.PlayTime을 읽기만 하도록 수정하였다.

---

## 2. MonsterManager 추가

새로운 MonsterManager를 제작하였다.

책임

- 살아있는 몬스터 관리
- Monster 등록
- Monster 제거
- AliveMonsterCount 제공
- OnAllMonstersDead 이벤트 제공

이를 통해 GameManager가
현재 몬스터 수를 직접 계산하지 않도록 변경하였다.

---

## 3. MonsterSpawner 수정

몬스터 생성 후

MonsterManager.RegisterMonster()

를 호출하도록 변경하였다.

기존에는 생성만 수행하였다면

이제는

생성
↓

MonsterManager 등록

까지 수행한다.

---

## 4. GameManager 리팩토링

GameManager가
게임의 승패를 판단하는 역할을 담당하도록 수정하였다.

현재 관리 이벤트

- PlayerStatus.OnDead
- GameTimer.OnStageTimeEnded
- MonsterManager.OnAllMonstersDead

승리 조건

Stage Time 종료

AND

Alive Monster == 0

↓

Game Clear

패배 조건

Player Dead

↓

Game Over

---

## 5. GameStateManager 확장

GameClear 상태를 추가하였다.

추가 내용

- GameClear Panel
- EnterGameClear()
- GameClear()
- GameState.GameClear

GameOver와 동일하게

- Cursor
- TimeScale
- UI

를 관리하도록 변경하였다.

---

## 6. MonsterSpawnSystem 리팩토링

기존

float gameTimer

삭제

GameTimer.PlayTime을 참조하도록 변경하였다.

시간 계산 책임을 제거하였다.

또한

ResetSpawnSystem()

을 추가하여
Spawn 관련 상태를 초기화할 수 있도록 수정하였다.

# 오늘 발생한 문제

리팩토링 이후
몬스터가 전혀 소환되지 않는 문제가 발생하였다.

원인

MonsterSpawnSystem과 MonsterSpawner의
SerializeField가 Inspector에서 연결되지 않은 상태였다.

GameTimer

MonsterManager

참조가 비어있어
Update() 초기에 return되고 있었다.

코드 문제가 아닌
Inspector 참조 문제였다.

# 느낀 점

이번 리팩토링을 통해

'누가 판단하는가'

를 중심으로 구조를 다시 설계하였다.

각 클래스가 직접 다음 행동을 수행하는 대신

이벤트를 발생시키고

GameManager가 최종 판단하도록 변경하였다.

앞으로 Boss, Wave, Object Pool 등을
추가하더라도 기존 구조를 크게 수정하지 않고
확장할 수 있는 기반이 마련되었다.
